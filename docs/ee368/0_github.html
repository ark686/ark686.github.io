Using GitHub ​

1. Overview ​

The various computers in the ECE Robotics Labs do NOT provide network-based synchronized files for each user. Files created and edited on one computer are not visible on the other computers (even for the same user id).

Two options to facilitate backup and transferring files:

  1. Use a USB drive.
  2. Use something like OneDrive.

A better solution is to use GitHub (https://github.com). GitHub is a web-based platform that uses Git, a version control system, to help developers store, manage, track changes, and collaborate on code projects. It acts as a central hub for hosting code repositories, offering features like bug tracking, task management, code reviews, and project planning, allowing teams and individuals to work on code in parallel and merge changes safely. .

This document provides the instructions on creating a GitHub account and using on Windows and Linux.

5. Linux - Access Repository ​

5.1 Personal Access Token ​

To access GitHub from Linux, need a Personal Access Token (cannot use a GitHub password):

  1. Sign in to your GitHub account on the GitHub website.
  2. Click your profile picture in the upper-right corner, then select Settings.
  3. In the left sidebar, scroll down and click Developer settings.
  4. In the left sidebar, under "Personal access tokens", select "Tokens (classic)"
  5. Select "Generate new token" and then "Generate new token (classic)"
  6. May have to enter your GitHub password
  7. Configure your token:
    • Note: Give the token a descriptive name
    • Expiration: Set an expiration date (for security, avoiding "No expiration" is recommended).
    • Select scopes: Check "repo"
    • Select the "Generate token" button at the bottom of the page.
  8. Copy the token immediately. For security, GitHub will not display the token again once you leave the page, so it must be stored in a secure location, like a password manager.

5.2 Clone Repository ​

  1. Create a suitable base directory (e.g., ~/git)
  2. Get the Clone link for the repository on GitHub, for example:

  1. Run the command, for example:

5.3 Push Changes to Repository ​

  1. To push changes to the GitHub repository, in the root directory of the repository on Linux (i.e., where the .git directory resides):
bash
git add -A
git commit
git push

Note that git commit will bring up a text editor for entering a description of the changes.

5.4 Pull Changes from Repository ​

  1. To get any changes from the GitHub repository, in the root directory of the repository on Linux (i.e., where the .git directory resides):
bash
git pull

Any changed files will be applied to your files on Linux. If there are no updates, should see a message of Already up to date.
