SDFormat ​

1. References ​

  • SDFormat, http://sdformat.org
  • XML, Extensible Markup Language, https://en.wikipedia.org/wiki/XML

2. Introduction ​

SDFormat (Simulation Description Format), sometimes abbreviated as SDF, is an XML format that describes objects and environments for robot simulators, visualization, and control.

3. Basic SDF World ​

This section defines a basic (empty) SDF world we can use as a starting point. The complete example is shown here:

xml
<?xml version="1.0" ?>
<sdf version="1.9">
    <world name="example_2_1">
        <physics name="1ms" type="ignored">
            <max_step_size>0.001</max_step_size>
            <real_time_factor>1.0</real_time_factor>
        </physics>
        <plugin
            filename="gz-sim-physics-system.so"
            name="gz::sim::systems::Physics">
        </plugin>
        <plugin
            filename="gz-sim-user-commands-system.so"
            name="gz::sim::systems::UserCommands">
        </plugin>
        <plugin
            filename="gz-sim-scene-broadcaster-system.so"
            name="gz::sim::systems::SceneBroadcaster">
        </plugin>

        <light type="directional" name="sun">
            <cast_shadows>true</cast_shadows>
            <pose>0 0 10 0 0 0</pose>
            <diffuse>0.8 0.8 0.8 1</diffuse>
            <specular>0.2 0.2 0.2 1</specular>
            <attenuation>
                <range>1000</range>
                <constant>0.9</constant>
                <linear>0.01</linear>
                <quadratic>0.001</quadratic>
            </attenuation>
            <direction>-0.5 0.1 -0.9</direction>
        </light>

        <model name="ground_plane">
            <static>true</static>
            <link name="link">
                <collision name="collision">
                <geometry>
                    <plane>
                    <normal>0 0 1</normal>
                    </plane>
                </geometry>
                </collision>
                <visual name="visual">
                <geometry>
                    <plane>
                    <normal>0 0 1</normal>
                    <size>100 100</size>
                    </plane>
                </geometry>
                <material>
                    <ambient>0.8 0.8 0.8 1</ambient>
                    <diffuse>0.8 0.8 0.8 1</diffuse>
                    <specular>0.8 0.8 0.8 1</specular>
                </material>
                </visual>
            </link>
        </model>
    </world>
</sdf>

3.1 Defining a World ​

Every SDF world starts with the tags shown below. Everything else goes inside the <world> tags.

xml
<?xml version="1.0" ?>
<sdf version="1.9">
    <world name="example_2_1">
    ...
    ...
    </world>
</sdf>

3.2 Physics ​

The physics tag specifies the type and properties of the dynamic engine. We chose the name “1ms” as the step size is 1 millisecond. The type is the type of the dynamic engine (physics library). There are options like, Ode, Bullet, Simbody and Dart. We set it to ignored, as choosing the type of the physics engine is not done through this tag yet.

xml
<physics name="1ms" type="ignored">
    <max_step_size>0.001</max_step_size>
    <real_time_factor>1.0</real_time_factor>
</physics>
1
2
3
4

<max_step_size> is the maximum time at which every system in simulation can interact with the states of the world. The smaller the value, the more accurate your calculations, but more computation power is needed. <real_time_factor> is the ratio of simulation time to real time.

3.3 Plugins ​

Plugins are a dynamically loaded chunk of code. For example, the “Physics” plugin is very important for simulating the dynamics of the world.

xml
<plugin
    filename="gz-sim-physics-system.so"
    name="gz::sim::systems::Physics">
</plugin>

3.4 Light ​

“Light” is an entity that defines light sources for a world (can have more than one).

xml
<light type="directional" name="sun">
    <cast_shadows>true</cast_shadows>
    <pose>0 0 10 0 0 0</pose>
    <diffuse>0.8 0.8 0.8 1</diffuse>
    <specular>0.2 0.2 0.2 1</specular>
    <attenuation>
        <range>1000</range>
        <constant>0.9</constant>
        <linear>0.01</linear>
        <quadratic>0.001</quadratic>
    </attenuation>
    <direction>-0.5 0.1 -0.9</direction>

3.5 Model – Ground Plane ​

Useful for the world to have a “Ground Plane”. This is added as a “model”.

xml
<model name="ground_plane">
    <static>true</static>
    <link name="link">
        <collision name="collision">
            <geometry>
                <plane>
                    <normal>0 0 1</normal>
                </plane>
            </geometry>
        </collision>
        <visual name="visual">
            <geometry>
                <plane>
                    <normal>0 0 1</normal>
                    <size>100 100</size>
                </plane>
            </geometry>
            <material>
                <ambient>0.8 0.8 0.8 1</ambient>
                <diffuse>0.8 0.8 0.8 1</diffuse>
                <specular>0.8 0.8 0.8 1</specular>
            </material>
        </visual>
    </link>
</model>
