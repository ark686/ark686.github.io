Build ROS 2 Packages ​

Taken from https://docs.ros.org/en/jazzy/Tutorials/Beginner-Client-Libraries/Creating-Your-First-ROS2-Package.html

1. Create a Python Package ​

First create a suitable ROS workspace source directory and change into it, for example:

bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/src

Create the ROS 2 package structure (replace my_node with the desired node name and replace my_package with the desired name of the package):

bash
ros2 pkg create --build-type ament_python --license Apache-2.0 --node-name my_node my_package

It should create the following directory structure and files:

text
my_package/
├── LICENSE
├── my_package
│   ├── __init__.py
│   └── my_node.py
├── package.xml
├── resource
│   └── my_package
├── setup.cfg
├── setup.py
└── test
    ├── test_copyright.py
    ├── test_flake8.py
    └── test_pep257.py

2. Build a Package ​

By having the packages in a ROS workspace directory structure, you can build all of the packages at the same time.

Change to the ROS workspace root directory, for example:

bash
cd ~/ros2_ws

Build your packages:

bash
colcon build --symlink-install

The --symlink-install option is used to create symbolic links from the install directory back to the source and build directories where possible. This feature is essential for a faster development cycle, as it allows changes to non-compiled files (like Python files, launch files, or configuration files) in the source space to be reflected immediately in the installed environment without requiring a full rebuild.

The output of the colcon build should be similar to:

text
Starting >>> my_package
Finished <<< my_package [0.96s]          

Summary: 1 package finished [1.15s]

The resulting directory structure should be similar to:

text
ros2_ws/
├── build
│   └── my_package
│       ├── my_package -> /home/ark686/ros2_ws/src/my_package/my_package
│       ├── my_package.egg-info
│       ├── prefix_override
│       │   └── __pycache__
│       ├── resource
│       └── share
│           └── my_package
│               └── hook
├── install
│   └── my_package
│       ├── lib
│       │   ├── my_package
│       │   └── python3.12
│       │       └── site-packages
│       └── share
│           ├── ament_index
│           │   └── resource_index

3. Source the Setup File ​

Open a new terminal and make sure the main ROS 2 setup file is sourced, for example:

bash
source /opt/ros/jazzy/setup.bash

Change directory to the ROS workspace root, for example:

bash
cd ~/ros2_ws

Then run:

bash
source install/setup.bash

Now that your workspace has been added to your path, you will be able to use your new package’s executables.

4. Use the Package ​

To run your package:

bash
ros2 run my_package my_node

The output should be:

text
Hi from my_package.

5. Customize package.xml ​

From ros2_ws/src/my_package, open package.xml in a text editor:

xml
?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
 <name>my_package</name>
 <version>0.0.0</version>
 <description>TODO: Package description</description>
 <maintainer email="ark686@todo.todo">ark686</maintainer>
 <license>Apache-2.0</license>

 <test_depend>ament_copyright</test_depend>
 <test_depend>ament_flake8</test_depend>
 <test_depend>ament_pep257</test_depend>
 <test_depend>python3-pytest</test_depend>

 <export>
   <build_type>ament_python</build_type>
 </export>
</package>

6. Customize setup.py ​

The setup.py file contains the same maintainer, maintainer_email and description as package.xml, so you need to set those as well. They need to match exactly in both files. The version and name (package_name) also need to match exactly, and should be automatically populated in both files.

python
from setuptools import find_packages, setup

package_name = 'my_package'

setup(
    name=package_name,
    version='0.0.0',
    packages=find_packages(exclude=['test']),
    data_files=[
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='ark686',
    maintainer_email='ark686@todo.todo',
    description='TODO: Package description',
    license='Apache-2.0',
    extras_require={
        'test': [
            'pytest',
        ],
    },
    entry_points={
        'console_scripts': [
            'my_node = my_package.my_node:main'
        ],
    },
)

7. Import Local Python Scripts ​

If you need to import local python scripts (*.py files residing in your package) to be called by your ROS nodes, you need to add a . prefix to the name so that they can be found when performing a ros2 run. For example:

python
import .my_helper_functions.py

8. Fix Issues with Building ​

If you encounter odd problems while doing a colcon build, you can try deleting the build directories and rebuilding.

Change to the root of your ROS 2 workspace, for example:

bash
cd ~/ros2_ws

Then run:

bash
rm -rf build log install
colcon build --symlink-install
source install/setup.bash
