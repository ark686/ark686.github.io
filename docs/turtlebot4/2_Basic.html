1. Accessing TurtleBot4 from ECE Laptops ​

Each TurtleBot4 (turtlebot4-XX.usask.ca, with XX = 01 to 11) is paired with a laptop (ece-2c95-XX.usask.ca, with XX = 01 to 11).

Log into a laptop using user "ece" and password "4thewin". Note that this is a shared login so you will have to copy your files to and from the laptop (suggest getting a USB stick). Also suggest creating a subdirectory named using your NSID and doing all your work there.

Perform the following command to set up access to the TurtleBot4:

bash
$ tb4

To check access (alias for 'ros2 topic list'):

bash
$ rtl

The output should look similar to the following (if a truncated list, try a couple of times, if it does not work, check out the Troubleshooting section):

4. TurtleBot4 Troubleshooting ​

4.1 WiFi Connectivity ​

Make sure that the appropriate TurtleBot4 is turned on. To turn on the TurtleBot 4, place it carefully into its dock until you see the LED on the dock light. The TurtleBot4 will take a couple of minutes to boot and you will hear a "trill" when it is done booting.

Check to see that all five green LEDs on the TurtleBot4 are on (if not all on and have just done a boot/reboot, wait another minute or two):

If the middle LED is off, that means that the TurtleBot4 does NOT have WiFi connectivity. Try Power Reset listed in the next section.

4.2 Power Reset ​

  1. Remove the TurtleBot4 from its dock.
  2. Hold down the large white button on the TurtleBot4 base until you hear a "trill" (takes a few seconds), this should power down the TurtleBot4.
  3. Place the TurtleBot4 back in its dock (make sure the LED on the dock lights). The TurtleBot4 should then reboot (takes a few minutes).

4.3 Cannot List ROS Topics ​

Make sure you have run "tb4" in your terminal session first.

If not getting the appropriate topics from "ros2 topic list" (rtl), first verify the laptop can "ping" the TurtleBot4:

bash
$ pingx
PING 10.136.206.230 (10.136.206.230) 56(84) bytes of data.
64 bytes from 10.136.206.230: icmp_seq=1 ttl=64 time=6.33 ms
64 bytes from 10.136.206.230: icmp_seq=2 ttl=64 time=10.1 ms
64 bytes from 10.136.206.230: icmp_seq=3 ttl=64 time=9.40 ms
64 bytes from 10.136.206.230: icmp_seq=4 ttl=64 time=12.2 ms

If you can "ping" the TurtleBot4 (which means you have WiFi connectivity), then reboot the Raspberry Pi (takes a couple of minutes) on the TurtleBot4 using:

bash
$ rebootx

If you cannot "ping" the TurtleBot4, then go through the Power Reset listed above.

4.4 Cannot List Battery State ​

If "ros2 topic list" is providing a valid list of ROS topics for the TurtleBot4 but there is no output from "rte /battery_state", that usually means that the Raspberry Pi on the TurtleBot4 is having trouble communicating with the TurtleBot4 base (different processor). You can try:

bash
$ restart_base

The TurtleBot4 base will take a couple of minutes to restart. The completion of the restart is indicated by a audible "trill". If that does not fix the issue of retrieving the /battery_state, then try the Power Reset listed above.
