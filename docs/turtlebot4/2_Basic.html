1. Accessing Turtlebot4 from ECE Laptops ​

Each Turtlebot4 (turtlebot4-XX.usask.ca, with XX = 01 to 11) is paired with a laptop (ece-2c95-XX.usask.ca, with XX = 01 to 11).

Log into a laptop and perform the following command to set up access:

tb4

To check access (alias for 'ros2 topic list'):

rtl

The output should look similar to the following (if a truncated list, try a couple of times):

/battery_state
/cmd_audio
/cmd_lightring
/cmd_vel
/cmd_vel_unstamped
/diagnostics
/dock_status
/function_calls
/hmi/buttons
/hmi/display
/hmi/display/message
/hmi/led
/imu
/interface_buttons
/ip
/joint_states
/joy
/joy/set_feedback
/odom
/parameter_events
/robot_description
/rosout
/scan
/tf
/tf_static
/wheel_status

To verify you can communicate to the base of the turtlebot4 (alias for 'ros2 topic echo --once'):

rte /battery_state

The output should look similar to the following (a percentage of 1.0 represents fully charged):

2. Turtlebot4 Commands ​

CommandDescription
dockDock Turtlebot4
pingxPing Turtlebot4
poPower Off Turtlebot4
rebootxReboot Raskberry Pi on Turtlebot4
restart_baseReboot Turtlebot4 base
ros2resetRestart ROS2 subsystem
rteros2 topic echo --once
rtlros2 topic list
scpxget [filename]Get [filename] from Turtlebot4
scpxput [filename]Write [filename] to Turtlebot4
sshxssh into Turtlebot4
sshxrun [cmd]Run [cmd] on Turtlebot4
tb4Set up access to Turtlebot4
undockUndock Turtlebot4
